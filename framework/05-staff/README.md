# Estágio 5 — Staff / Principal (Specialization, Influence, Public Output)

## Por que esse estágio existe

O framework original parava no Senior. Senior generalista é forte: arquiteta sistemas distribuídos, lê DDIA, escreve ADRs, lidera projeto. Mas há um eixo distinto que separa Senior consolidado de Staff/Principal real — e pular isso significa ficar Senior pra sempre.

A diferença é qualitativa: Staff/Principal **multiplica via influência** (não só via código), **especializa** (ou se posiciona como T-shape forte), **publica** (audiência além do empregador), **mentoraz** (outros crescem por você), **lê paper** (não só blog), **constrói from scratch** (entende abaixo da linha de abstração), **fala org** (Conway, team topologies), **pensa business** (unit economics), e **escolhe trajetória** (specialist deep ou generalist com peso).

Este Estágio 5 é o **gap closer entre Senior consolidado e Staff/Principal market-ready**. Não é "mais um capstone"; é a integração + posicionamento de tudo o que veio antes, com módulos que cobrem dimensões fora do que código sozinho ensina.

**Promessa de saída do Staff:** você consegue pleitear título Staff em empresas tier-1 com evidência multidimensional — código, escrita pública, mentoria, comunidade, papers, e um eixo de profundidade declarado. Não é garantia de comp; é garantia de **estar à altura**.

**ST07-ST10 são opcionais — specialty tracks**. Faça apenas se eixo de carreira aponta:
- ST07 Embedded: IoT, robotics, automotive, hardware-software co-design.
- ST08 Hardware Design: profundidade em devices, hardware startup, ASIC/FPGA.
- ST09 Bioinformatics / Scientific Computing: biotech, pharma, healthcare AI, climate, research.

Pra 95% das trajetórias SaaS / cloud / web, ST07-ST10 são **skip seguro**. Inclusos pra completar cobertura.

Os outros 6 (ST01-ST06) são **núcleo**. Pode atacar em paralelo:
- ST01 + ST04 são heavy lift técnico. Atacar weekends durante meses.
- ST02 é série de capstones; cada um 6-10 semanas.
- ST03 + ST05 + ST06 são mais "running disciplines" — começam logo, mantêm cadência.

## Capstone

[CAPSTONE-staff.md](CAPSTONE-staff.md) — **Especialização + Portfolio + Promo Case**.

Pick uma das 6 tracks (Distributed Systems, Platform, Frontend, Data/ML, Security, Founding) e produza output integrando tudo do framework + ST01-ST07. Output: track-specific capstone showcase, portfolio site, promo case, 1 talk, OSS lib com tração, paper habit consolidada, 3 mentees acompanhados.

Esse é o **fim do framework**. Não há Estágio 6.

---

## Postura recomendada para este estágio

- **Specialization é escolha ativa**. Default = generalist; declare specialist se deliberada.
- **Public output não é vanity**. Write-to-think + audience compounds.
- **Mentoria não é favor — é prática**. Time budget allocated, disciplined.
- **Paper reading sustainable**. 1/semana, 12 semanas mínimo pra forma habit.
- **Promo case é currículo do trabalho técnico**. Document evidence à medida que produz.
- **Multi-domain capstones forçam transferência**. Stay out of comfort zone Logística.
- **Conway's Law é alavanca**. Org design importa tanto quanto code.
- **Build-from-scratch demystifica**. Construa pelo menos 2 dos 5 toys.

## Sobre cobiça e sobre cuidado

Staff/Principal é **lugar honesto**, não tier de status. Se você faz Stage 5 só pra "ter título Staff", o output será raso. Se você faz pra desenvolver-se nessas dimensões reais (clarity de pensamento via writing, multiplicação via mentoria, depth via build-from-scratch), o título é consequência.

Igualmente: nem todo mundo precisa ir pra Staff. Senior IC com vida cheia fora trabalho é trajetória válida. Staff demanda investment alto. Decida com sobriedade.

---

## Tempo? Não — outra vez

Como demais estágios: **leva o que precisar**. Mas atenção: 6 módulos núcleo + capstone é 12-24 meses de trabalho real, depois de Senior completo. Esse é o estágio mais longo. Comprometa-se com cadência, não calendário.

---

## Próximo passo (depois de Staff)

Não há "Stage 6". As trajetórias divergem:
- Continue Staff em empresa atual e cresça scope.
- Move pra Staff em empresa-alvo (Stripe, Anthropic, Cloudflare, etc.).
- Promote-se a Principal / Distinguished se shape fit.
- Switch pra manager se appeals.
- Founding engineer ou empresa própria.
- Sabbatical / OSS full-time se possível.

Framework te entregou ferramentas; do Staff em diante, é sobre **que problemas você quer atacar**.
